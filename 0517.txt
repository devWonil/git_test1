HTML 은 Hyper Text Markup Language의 약자입니다

Hyper란

뛰어넘다, 초월하다 의미를 가지고 있다
text 정보를 순차적으로 접근하며, 정보를 순차적으로 습득한다 (책)

Hyper Text
어떠한 정보에 대한 접근을 순차적인 접근 방식을 뛰어넘는다는 의미를 가진다

Markup
어디든 간에 Mark, 즉 표시를 해두는 것을 의미한다

표시가 없을 때

Html 이란 Hyper Text Markup Language의 약자입니다. 웹&앱 개발과정 D 강의장에서 진행합니다.

표시가 있을 때
- 제목 : Html 이란
- 내용 : Hyper Text Markup Language의 약자입니다.
- 위치 : 웹&앱 개발과정 D 강의장에서 진행합니다.

즉, 문서를 구조적으로 표현하기 위한 것이 마크업의 개념입니다

브라우저의 구성요소는 모든 게 content입니다
브라우저는 컴퓨터와 마찬가지로 인간의 언어를 이해할 수 없기 때문에 모든 것을 알려줘야 한다. 이건 title이야, 이건 이미지야 이건 날짜야 등 모든 것을 알려줘야 한다.

즉, 브라우저에게 우리가 사용할 content를 알려 줘야 하기 때문에 사용하는 언어가 html 이다.

- 안드로이드 UI를 그리기 위해 특화된 언어이다

dp와 px의 개념
핸드폰마다 크기가 다른데 어떻게 화면을 그려야 할까???

핸드폰은 정말 다양한 화면의 크기와, 해상도가 존재하는데 내가 원하는 UI를 어떻게 같은 느낌, 같은 모습으로 그려야 하나?

- 픽셀, DPI, DP 단위

픽셀 (핸드폰 화면에 빛이 나오는 전구 -> 가장 작은 단위)
해상도 (고해상도) : 정해진 수치 안에서 전구가 많을 수록 고해상도라고 합니다.
정해진 수치를 DPI 라고 한다.

DPI (Dot Per Inch)
 - ldpi -> (1인치에 120px)
 - mdpi -> (1인치에 160px)
 - hdpi -> (1인치에 240px)
 - xhdpi -> (1인치에 320px)
 - xxhdpi -> (1인치에 480px)
 - xxxhdpi -> (1인치에 640px)

안드로이드에서는 dp라는 단위를 사용한다 (가상단위)
100dp (내가 알아서 계산해서 적절한 크기로 그려줄게)

dp (Density Independent Pixel) 픽셀에 독립적인 단위

px : dp * 단말 dpi / 기본 160
dp : px * 기본 160 / 단말 dpi

25dp = ( 100px * 160 / 640)
50dp = ( 100px * 160 / 320)

결론 : 안드로이드에서는 dp라는 단위를 사용하면 해상도에 따라서 픽셀값을 자동으로 계산을 해주고 화면에 그려준다
